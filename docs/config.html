FMEasyCalendar: Setup and Configuration

To configure your database for use with FMEasyCalendar, following these instructions.

1. Copy (or import) the EasyCalendar table from the demo file.

The EasyCalendar table is used to generate and display the calendar. No changes to the table are necessary.

2. Import the FMEasyCalendar script into your database.

You will likely get a message indicating that four layouts and one field that are referenced in the script are missing. These errors are caused by references that the demo database uses to display the calendar in "day mode." You can either ignore these errors, disable the script steps that are used to display the calendar in day mode (the steps in the "If" block located at around line 38 of the script), or add your own script steps.

3. Configure EasyCalendar using the settings in the FMEasyCalendar script.

The settings that are supported follow:

  • $$Event_Table_Name: The name of the table from which events are to be pulled. Note that you can pull from multiple tables by using a return-separated list of table occurrences.
  • $$Event_Begin_Date_Field_Name: The field that should be used as the "begin date" for the events. Return-separate the values if you are pulling from multiple tables.
  • $$Event_End_Date_Field_Name: The optional field to use as the "end date" for the events. Return-separate the values if you are pulling from multiple tables.
  • $$Event_Begin_Time_Field_Name: The optional field to use as the "begin time" for the events. Return-separate the values if you are pulling from multiple tables.
  • $$Event_End_Time_Field_Name: The optional field to use as the "end time" for the events. Return-separate the values if you are pulling from multiple tables.
  • $$Event_Desc_Field_Name: The field from which the name or description of the events should be pulled. Return-separate the values if you are pulling from multiple tables.
  • $$Event_Color_Field_Name: The optional field to use to determine the text color of the events. Return-separate the values if you are pulling from multiple tables. Values should be RGB values in this form: 0;0;255
  • $$Event_Style_Field_Name: The optional field to use to determine the text style of the events. Return-separate the values if you are pulling from multiple tables. Supported values include: Plain, Bold, Italic, Underline, HighlightYellow, Condense, Extend, Strikethrough, SmallCaps, Superscript, Subscript, Uppercase, Lowercase, Titlecase, WordUnderline, DoubleUnderline, AllStyles.
  • $$Event_Filter: An optional filter to be applied when events are selected from the table, in SQL form. Return-separate the values if you are pulling from multiple tables.
  • $$Start_Weeks_On: The day of the week (Sunday or Monday) that calendars should start on.
  • $$Layout_Name_Month_View: The name of the layout to use when displaying the calendar in a monthly format.
  • $$Layout_Name_Week_View: The name of the layout to use when displaying the calendar in a weekly format.
  • $$Time_Format: The format to use (standard or military) when displaying event times.
  • $$Use_PSoS: Indicates whether or not to use the "Perform Script on Server" (PSoS) script step, which can improve performane in hosted solutions. A value of 1 indicates that PSoS should be used (when available), while 0 indicates that it should now be used.

4. Add an "EasyCalendar - Month View" layout to your database.

After adding the EasyCalendar table, FileMaker Pro should have automatically created an "EasyCalendar" layout for you. If so, you can simply rename that layout to: EasyCalendar - Month View

The layout will only be viewed in List View. Therefore, if you would like, you can use the Layout Setup options to disable Form View and Table View.

5. Copy the layout objects from the "EasyCalendar - Month View" layout in the demo database to the "EasyCalendar - Month View" layout in your database.

Be careful when positioning layout objects, so that they end up in the correct layout parts (Header, Body, etc).

6. Optional. Setup an "EasyCalendar - Week View" layout.

To add support for displaying the calendar in a weekly format, setup a layout in much the same way that you did above (for the "Month View"). Name the layout "EasyCalendar - Week View." Then copy the layout objects from the "EasyCalendar - Week View" layout in the demo database to the "EasyCalendar - Week View" layout in your database. As before, be careful when positioning layout objects, so that they end up in the correct layout parts (Header, Body, etc).

7. Optional. Setup privileges on the EasyCalendar table.

To prevent users from seeing the other users' calendars, you might want to setup the privileges on the EasyCalendar table so that the ability to view records in the table is "limited" and based on this calculation: Owner_Account_Name = Get ( AccountName ). For an example, refer to the demo database, and specifically to the "Employees" privilege set.

To display the calendar, you can simply run the FMEasyCalendar script.

The script supports the following parameters:

  • $$display_date: The start date for the calendar to be displayed. The default value is the current date.
  • $$display_mode: The mode (or type) of calendar to be displayed. Supported values include month, week, and day. The default is "month."

When passing parameters to the script, separate them using a return character.
