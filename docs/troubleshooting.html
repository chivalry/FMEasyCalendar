FMEasyCalendar: Troubleshooting Tips

Having problems with FMEasyCalendar? Included below are answers to some of the more common questions that we've received, and tips on how to resolve certain issues.

My calendar is displaying question marks instead of the event information.

Double-check the settings that you are using for the table names and fields that you are trying to pull event info from. Specifically, look at $$Event_Table_Name (the name of the table that the events are stored in), $$Event_Begin_Date_Field_Name (the field in which the event dates are stored), and so on. For the $$Event_Table_Name setting, make sure that you are using a valid table occurrence name.

Note: Also check the value of the $$Event_Color_Field_Name setting. In the demo database, this is set to "EzCal_Text_Color," which is a field in the Contacts table. If you do not have a field that you want to use to specify event colors, simply clear this setting.

Layout objects that I've pasted from the demo database are broken.

Be sure to setup the EasyCalendar table in your database before attempting to setup the calendar layouts. Otherwise, references to fields will be break.

Script steps in the FMEasyCalendar script are broken.

There are approximately 5 steps that refer to layouts and fields in the demo database, which will likely end up broken when you paste the script into your database. Those steps are in the area of script near the "If a day view was requested..." comment. If you want to add support for a day view in your database, you will need to adjust those steps accordingly.

In the month view of my calendar, only the first week is displayed correctly. Other weeks are greyed out.

To fix this, disable the “Delineate fields in current record only" option (which is available as a Layout Setup option).

In the month view of my calendar, the first row (or selected record) shows with a blue background, but the others do bot.

To correct this, open the layout in Layout mode, double-click the Body part of the layout, and disable the “Use active row state" option.

The event text styling options that I am using aren't working.

First, confirm that the field that you want to use to style the events is setup properly in EasyCalendar's settings. The $$Event_Style_Field_Name setting should be set to the name of the field that contains the styling information that you want to apply to an event.

EasyCalendar supports the same styling options that FileMaker's "TextStyleAdd" function supports, including: Plain, Bold, Italic, Underline, HighlightYellow, Condense, Extend, Strikethrough, SmallCaps, Superscript, Subscript, Uppercase, Lowercase, Titlecase, WordUnderline, DoubleUnderline, and AllStyles. To apply multiple styles, using the "+" operator between individual styles. For example: Bold+Italic

The event text color options that I am using aren't working.

First, confirm that the field that you want to use to apply colors to events is setup properly in EasyCalendar's settings. The $$Event_Color_Field_Name setting should be set to the name of the field that contains the color information that you want to apply to an event.

To specify a color, set the field to its RGB color triplet, in this format: red;green;blue. The values for red, green, and blue should be set to numbers between 0 and 255. For assistance in selecting an RGB color value, we recommend using the Color Schemer Web site.

Note: When specifying an RGB color, be sure to only include the RGB values, separated by semi-colons. For example: 255;0;0. Do not include the "RGB" function in the value.

The event text styling and color options are working perfectly in FileMaker Pro and Go, but not in FileMaker WebDirect.

This is a known issue, and it seems to be a limitation of WebDirect.
